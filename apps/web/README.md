# МышМат — веб-приложение (Next.js)

Главный экран ребёнка и переход к заданиям предмета. Свёрстано на основе
`myshmat-home_5.html`, разложено на React-компоненты, типизировано под ТЗ v1.

## Запуск

```bash
cd apps/web
npm install
npm run dev
# http://localhost:3000
```

## Логика МышРутки (ядро ТЗ v1)

Реализована в `types/domain.ts`:

- Предмет «сдан со стороны ребёнка» при статусе `submitted | successful | perfect`.
- `isMyshroutkaEarned(subjects)` → `true`, когда ВСЕ 4 предмета сданы.
- При этом `session.myshroutkaGranted = true`, и `buildWorldState()` открывает
  **одним событием**: Карту тем, Лайфхаки, Сундуки, Награды.
- Дуэли (`DUELS_ENABLED_MVP = false`) на MVP всегда «Скоро» — карточка-заглушка.

Пока `myshroutkaGranted === false`, карточки игрового мира получают класс
`.is-locked` (блюр + подпись «Откроется после Daily»), клики отключены.

## Экран задания — поведение (обновлено)

- **Маскот убран** — задание во всю ширину (место под длинные условия).
- **Подсказка только со 2-й попытки**: на первой попытке кнопки подсказки нет;
  появляется после первой ошибки. Метрика самостоятельности.
- **Многошаговые задания** (platform): шаги `reading` (текст, только «Далее»)
  и `question` (вопрос). Навигация вперёд/назад; можно идти дальше даже с ошибкой.
  Прогресс — «Шаг N из M» текстом.
- **Аналитика по шагам** (`StepStat`): attempts, hintUsed, solvedFirstTry,
  skippedWithError. `computeAutonomy()` → доля шагов, решённых сам с 1-й попытки.
  Сейчас отчёт пишется в console.log на завершении; на проде → POST в API.

## Шаги 1–3 (реализовано)

### Шаг 1 — Бэкенд (Supabase)
- `packages/db/0001_init.sql` — схема: профиль, задания+шаги, Daily-сессии,
  попытки, пошаговая аналитика (`task_step_stats`), награды, наклейки.
  Функция `recompute_daily_and_grant_myshroutka()` выдаёт МышРутку, когда все
  предметы дошли до submitted.
- `packages/db/0002_seed.sql` — демо-ребёнок Артём + банк заданий (как в моках).
- `packages/db/0003_auth.sql` — поля входа ребёнка (pin_hash, short_code) + 2-й ребёнок Маша. Нужна для `/api/login`.
- `src/lib/supabase.ts` — серверный клиент (null, если env не задан).
- `src/lib/data-db.ts` — выборки из БД с тем же контрактом, что у моков.
- `src/lib/data.ts` — единый фасад: БД, иначе моки. Все страницы используют его.
- `src/app/api/attempts/route.ts` — POST попытки + пошаговой аналитики,
  вызывает RPC выдачи МышРутки.
- Настройка: скопировать `.env.example` → `.env.local`, заполнить ключи Supabase,
  выполнить SQL-миграции. Без env — приложение работает на моках.

### Шаг 2 — Олимпиадный экран
- `/olympiad/[id]` — фокус на ЗАПИСИ решения: что дано → план → шаги →
  рассуждение → ответ (`SolutionRecord`). Прогресс полноты записи,
  подсказка со 2-й попытки. Метрика `reasoningCompleteness()`.

### Шаг 3 — Доводка экранов
- `/revisions` — список доработок с фидбеком методиста, ведёт на задание.
- `/stickers` — альбом наклеек (сетка 120, собрано 23).
- `/topics` — карта тем (олимпиадный маршрут), острова ведут на `/olympiad/[id]`.
- Трекер чтения 30 минут (`ReadingTimer`) — встроен в «Дневник читателя».
- Заглушки `/lifehacks`, `/chests`, `/rewards` — «скоро».

## Роли и дополнительные экраны (новое)

### Вход ребёнка — `/login`
Выбор профиля плиткой → ввод PIN (4 цифры) на экранной клавиатуре.
API `/api/login` (режимы `pin` и `code`). На моках PIN: Артём `1234`, Маша `0000`.
На проде PIN сравнивается через bcrypt, короткий код — по `child_profiles.short_code`.

### Панель методиста — `/methodist`
Очередь работ «на листочке». По каждой: фото решения + комментарий ребёнку +
вердикт: ⭐ Отлично (`perfect`) / ✓ Зачёт (`successful`) / ↩ На доработку (`needsRevision`).
API `/api/review` обновляет попытку, прогресс предмета и пересчитывает МышРутку
(зачёт последнего предмета может открыть игровой мир).

### Панель родителя — `/parent`
Карточки детей: Daily за неделю, **самостоятельность** (доля шагов решённых сам
с 1-й попытки без подсказки), звёзды, статус предметов, лента активности.

### Экран чтения
Текст → вопросы шагами. На шагах-вопросах текст доступен по кнопке
«📖 Перечитать текст» (collapsible), чтобы не терять контекст.
