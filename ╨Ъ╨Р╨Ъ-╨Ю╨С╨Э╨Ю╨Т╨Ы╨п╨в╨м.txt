КАК БЫСТРО ОБНОВЛЯТЬ И ЗАЛИВАТЬ ПРОЕКТ
========================================

В папке проекта есть два помощника:

  update.bat — обновляет файлы проекта из скачанного архива
  push.bat   — заливает изменения в GitHub

--------------------------------------------------
ОБЫЧНЫЙ ЦИКЛ (когда я дал новый архив)
--------------------------------------------------

1. Скачайте новый myshmat-nextjs.tar в папку Загрузки
   (качайте как обычно — имя вида .tar, .tar_1, .tar_2 не важно,
    скрипт сам возьмёт самый свежий).

2. Двойной клик по  update.bat
   - он найдёт свежий архив в Загрузках,
   - распакует поверх проекта,
   - папку .git (историю) не тронет.
   Подтвердите нажатием Y.

3. Двойной клик по  push.bat
   - покажет, что изменилось,
   - спросит описание (можно просто Enter),
   - сам сделает add + commit + push.

Готово. Vercel подхватит изменения сам.

--------------------------------------------------
ЕСЛИ ЧТО-ТО НЕ ТАК
--------------------------------------------------

* "Не нашёл myshmat-nextjs.tar*"
  -> Проверьте, что архив скачан в папку Загрузки.
     Если качаете в другое место — откройте update.bat
     Блокнотом и поправьте строку:  set "DOWNLOADS=..."

* При push просят логин/пароль
  -> Логин: ksmishukova-alt
     Пароль: ваш Personal Access Token (не пароль от GitHub).
     Чтобы не спрашивал каждый раз, один раз выполните в терминале:
       git config --global credential.helper manager

* Хотите проверить, что реально изменилось перед заливкой
  -> В push.bat это видно в блоке "Текущие изменения".

Эти три файла (update.bat, push.bat, КАК-ОБНОВЛЯТЬ.txt) в репозиторий
не попадают — они только для вашего компьютера.
