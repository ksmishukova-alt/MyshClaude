# МышМат — handoff

> **Полное видение, архитектура всех flow, модель данных, стек и дорожная карта —
> в `plan.md`** (построен на глубоком опросе). Этот файл — краткая выжимка для быстрого старта.

Образовательная платформа для детей 8–13 лет. Маскот — мышь в синей толстовке.
Стек: Next.js 15 + React 19, Postgres (Supabase → позже self-hosted HostVDS), bcryptjs.

## Цель продукта

Развитие мышления школьников через **олимпиадную математику** — чтобы дети мыслили
системно и нестандартно и умели решать любые задачи и проблемы.

**Ядро продукта — игровой мир** с темами по олимпиадной математике, разделёнными на
уровни сложности **L1–L5** (от пошагового, с поддержкой, до олимпиадно-самостоятельного).
Продвижение по уровням развивает способность рассуждать, искать неочевидные ходы и
доводить решение до конца.

**Daily — это надстройка-вход, а не основа.** Ежедневный Daily по 4 предметам
(математика, русский, чтение, английский) служит проверкой готовности и поддержанием
базовых навыков. После выполнения всех 4 предметов (статус `submitted`) ребёнок получает
МышРутку, которая открывает доступ к игровому миру. То есть Daily — это «ворота» ко входу
в основную, олимпиадную часть, а не центр продукта.

Задания Daily решаются интерактивно прямо на платформе; письменные и аудио — с загрузкой
фото/записи и ручной проверкой методистом.

## Что уже сделано

### Интерактивные раннеры (10 механик)
Все 30+ типов заданий из методичек сведены к 10 переиспользуемым раннерам
(в `apps/web/src/components/runners/`):
- **punctuation** — расстановка знаков и заглавных букв.
- **order** — сбор предложения/текста из карточек (drag + клик).
- **wordfix** — замена неподходящего по смыслу слова.
- **gapinput** — ввод в пропуски (буквы, слова, анаграммы, проверочные слова).
- **sort** — распределение по 2–4 колонкам.
- **fields** — разбор по составу / словосочетание (подписанные поля).
- **audio** — аудиодиктант: плеер (лимит 2 прослушивания) + фото. Ручная проверка.
- **listening** — аудирование: диалог/текст + вопросы (варианты и короткий ввод). Автопроверка.
- **proofread** — корректура: найти и исправить ошибки кликом по словам. Автопроверка.
- **readaloud** — чтение вслух с реальной записью звука (MediaRecorder API). Ручная проверка.

Поток проверки: solving → [Проверить] → correct / wrong (до 3 попыток, попытка N из 3 + подсказка)
→ failed («Пока не получилось» → в доработки). `audio` и `readaloud` — ручные, без кнопки «Проверить».

### Игровой мир (ядро продукта) — текущее состояние
- Олимпиадный маршрут «островов» (`/topics`) с состояниями locked/open и переходом
  «Вперёд!» к первому доступному острову.
- Экран решения олимпиадной задачи (`/olympiad/[id]`) фокусируется на **записи рассуждения**,
  а не только на ответе: ребёнок проходит этапы что дано → план → шаги → рассуждение → ответ
  (тип `OlympiadProblem` + `SolutionRecord` в `types/domain.ts`).
- Пока заведено лишь несколько демо-задач (`logic`, `parity`) в `mock-data.ts`.
- **Система уровней L1–L5 ещё НЕ формализована в коде** — это ключевая задача (см. ниже).

### Daily (надстройка-вход)

## Проверки качества (перед каждой упаковкой)
- Проверка типов (tsc со stub-декларациями; известное ложное срабатывание про `key` в DailyCard).
- Аудит импортов/экспортов и баланса скобок.
- Для punctuation: длина `expectedMarks` == числу слов, индексы `expectedCapitals` в границах.
- Для proofread: индексы `proofFixes` указывают на реально ошибочные слова.
- ASCII-чистота `update.bat` / `push.bat` (кириллица ломает выполнение).

## Следующие шаги

### Приоритет 1 — олимпиадное ядро (игровой мир)
1. **Формализовать уровни L1–L5** в типах и данных: от пошагового решения с поддержкой (L1)
   до олимпиадно-самостоятельного (L5). Описать, чем отличается каждый уровень
   (объём подсказок, степень самостоятельности записи рассуждения, сложность задач).
2. **Спроектировать карту тем** олимпиадной математики (логика, чётность, инварианты,
   комбинаторика, графы, принцип Дирихле и т.д.), разбить каждую тему по уровням L1–L5.
3. **Наполнить банк олимпиадных задач** с этапами рассуждения (что дано → план → шаги →
   рассуждение → ответ), привязав к темам и уровням.
4. Связать прогресс по уровням с открытием новых островов/тем игрового мира.

### Приоритет 2 — Daily (вход) и инфраструктура
5. **Аудиофайлы.** Записать/получить mp3 для русских диктантов и английского аудирования,
   проставить в поле `audioUrl` соответствующих заданий. (ElevenLabs в среде разработки недоступен.)
6. **Перенос в боевую БД.** SQL-сид контента (Daily недель 1–10 + олимпиадные задачи)
   с привязкой Daily к реальным датам методического плана (22.06–28.08.2026), вместо моков.
7. **Боевая проверка.** Прогнать `setup_all.sql` в Supabase, выполнить `npm install`,
   проверить сквозной сценарий: вход ребёнка → Daily → submitted → МышРутка → игровой мир → задача.
8. **Выверка контента** через `/preview` пользователем; правки по обратной связи.
9. После выверки — скрыть `/preview` на боевой версии.
